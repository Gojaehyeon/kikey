# 🐱 Kikey

> Cute keyboard sound app for Mac. Type happily, work calmly.

[![GitHub release](https://img.shields.io/github/v/release/Gojaehyeon/kikey?include_prereleases&label=release)](https://github.com/Gojaehyeon/kikey/releases)

Kikey turns every keystroke into a satisfying little sound — meows, mechanical clicks, typewriter taps, water drops, bell chimes, heartbeats, and more. It lives in your menu bar (or hides entirely), uses zero‑lag native audio, and never records what you type.

> Kikey is built around one rule: **typing should make you smile, never spy on you.**

## Features

- 🎵 **8 built‑in sound packs** — all procedurally synthesized, zero audio files
- 🪶 **~400KB total** binary, no third‑party dependencies
- ⚡️ **Zero‑lag** native Swift / `AVAudioEngine` with 16‑voice polyphony
- 🎲 **Randomized pitch & velocity** per keystroke for a natural feel
- 🤫 **Press / release sound separation**
- 👻 **Hide menu bar icon** entirely — control via global hotkey only
- 🔒 **Secure Input aware** — automatically silent in password fields, sudo prompts, lock screens
- 🚀 **Launch at login** (toggleable)
- 🛡 **Privacy‑first** — keystrokes are processed locally; nothing is logged or transmitted (see [SECURITY.md](SECURITY.md))
- 🍎 macOS 14 (Sonoma) or later, Apple Silicon and Intel

## Install

### Download a release

Grab the latest `Kikey-X.Y.Z.dmg` from [GitHub Releases](https://github.com/Gojaehyeon/kikey/releases). Drag `Kikey.app` to `/Applications`. On first launch, right‑click → **Open** to bypass Gatekeeper (until the build is notarized).

### Build from source

```bash
brew install xcodegen
git clone https://github.com/Gojaehyeon/kikey.git
cd kikey
make build       # generates Kikey.xcodeproj and builds Release
make run         # builds and launches the .app
make dist        # produces dist/Kikey-X.Y.Z.{zip,dmg}
make icon        # regenerate AppIcon from SF Symbol
```

The Xcode project is generated by [XcodeGen](https://github.com/yonaskolb/XcodeGen) from `project.yml` and is `.gitignore`d. If you add a Swift file under `Sources/Kikey/`, just run `make project` again.

## Usage

1. Launch Kikey — it appears in the menu bar (🐱).
2. Grant **Input Monitoring** when prompted (System Settings → Privacy & Security → Input Monitoring).
3. Pick a sound pack from the menu.
4. Type. Smile.

### Hotkeys

| Shortcut | Action |
|---|---|
| ⌘⇧K | Toggle Kikey on/off |

### Hiding the menu bar icon

Open Kikey → uncheck **Show menu bar icon**. Kikey keeps running and you can toggle it via ⌘⇧K. To re‑show the icon, relaunch Kikey from `/Applications` and re‑enable it in Settings.

## Privacy

See [SECURITY.md](SECURITY.md). The short version: Kikey reads only the **keycode** of each key event so it can play a sound. Nothing is stored on disk, nothing leaves your Mac. When `IsSecureEventInputEnabled()` is true, Kikey is automatically silent.

## Architecture

```
Sources/Kikey/
├─ KikeyApp.swift              SwiftUI @main, MenuBarExtra
├─ AppDelegate.swift           NSApplicationDelegate hooks
├─ Settings.swift              @Observable settings, persisted via UserDefaults
├─ AudioEngine.swift           AVAudioEngine + 16-voice polyphony pool
├─ KeyEventTap.swift           CGEventTap wrapper
├─ SecureInputGuard.swift      IsSecureEventInputEnabled() helper
├─ HotKeyManager.swift         Carbon RegisterEventHotKey wrapper
├─ LaunchAtLoginManager.swift  SMAppService wrapper
├─ SoundPack.swift             SoundPack protocol + Synth helpers
├─ SoundPacks/
│  ├─ SoundPackRegistry.swift
│  ├─ CatPack.swift            🐱 cat meow synth
│  ├─ MechPack.swift           ⌨️ mechanical click
│  ├─ TypewriterPack.swift     📝 typewriter strike
│  ├─ PopPack.swift            🫧 bubble pop
│  ├─ DropPack.swift           💧 water drop
│  ├─ BellPack.swift           🔔 glass bell chime
│  ├─ HeartPack.swift          💗 warm heartbeat
│  └─ ClickPack.swift          🖱️ mouse click
├─ MenuBarView.swift           SwiftUI menu bar content
├─ SettingsView.swift          SwiftUI Settings scene
└─ PermissionsView.swift       Onboarding for Input Monitoring
```
