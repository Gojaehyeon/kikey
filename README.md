# 🐱 Kikey

> Cute keyboard sound app for Mac. Type happily, work calmly.

Kikey turns every keystroke into a satisfying little sound — meows, mechanical clicks, typewriter taps, water drops, and pops. It lives in your menu bar (or hides entirely), uses zero‑lag native audio, and never records what you type.

## Features

- **5 built‑in sound packs** — Cat 🐱, Mech ⌨️, Typewriter 📝, Pop 🫧, Drop 💧
- **Procedurally synthesized audio** — no bundled wav files, instant load, tiny binary
- **Press / release sounds** with randomized pitch + velocity for a natural feel
- **Polyphonic** — fast typing never drops a note
- **Hide menu bar icon** option — control via global hotkey only (⌘⇧K)
- **Secure Input aware** — automatically silent in password fields
- **Launch at login** (toggleable)
- **Privacy‑first** — keystrokes are processed locally and never logged or transmitted
- **Native Swift / SwiftUI**, ~zero CPU when idle, runs on Apple Silicon and Intel

## Requirements

- macOS 14 (Sonoma) or later
- Input Monitoring permission (granted on first launch)

## Build from source

```bash
brew install xcodegen
make build       # generates Kikey.xcodeproj and builds Release
make run         # builds and launches the .app
make open        # opens the project in Xcode
```

The Xcode project is generated by [XcodeGen](https://github.com/yonaskolb/XcodeGen) from `project.yml` and is `.gitignore`d.

## Usage

1. Launch Kikey — it appears in the menu bar (🐱).
2. Grant Input Monitoring when prompted (System Settings → Privacy & Security → Input Monitoring).
3. Pick a sound pack from the menu.
4. Type. Smile.

### Hotkeys

| Shortcut | Action |
|---|---|
| ⌘⇧K | Toggle Kikey on/off |

### Hiding the menu bar icon

Open Settings → General → uncheck **Show menu bar icon**. Kikey keeps running; toggle it via ⌘⇧K and re‑show the icon by relaunching from `/Applications`.

## Privacy

Kikey uses a `CGEventTap` to observe key‑down and key‑up events so it can play a sound. It only reads the **keycode** (which physical key) — never the character, never the modifiers in plaintext, never the focused application's content. Nothing is written to disk, nothing is sent over the network. The source is open; verify it yourself in `Sources/Kikey/KeyEventTap.swift`.

When `IsSecureEventInputEnabled()` is true (e.g. you focus a password field, sudo prompt, lock screen), Kikey is automatically silent.

## Architecture

```
Sources/Kikey/
├─ KikeyApp.swift              SwiftUI @main, MenuBarExtra
├─ AppDelegate.swift           NSApplicationDelegate hooks
├─ Settings.swift              @Observable settings, persisted via UserDefaults
├─ AudioEngine.swift           AVAudioEngine + voice pool
├─ KeyEventTap.swift           CGEventTap wrapper
├─ SecureInputGuard.swift      IsSecureEventInputEnabled() helper
├─ HotKeyManager.swift         Carbon RegisterEventHotKey wrapper
├─ LaunchAtLoginManager.swift  SMAppService wrapper
├─ SoundPack.swift             SoundPack protocol + voice scheduling
├─ SoundPacks/
│  ├─ SoundPackRegistry.swift
│  ├─ CatPack.swift
│  ├─ MechPack.swift
│  ├─ TypewriterPack.swift
│  ├─ PopPack.swift
│  └─ DropPack.swift
├─ MenuBarView.swift           SwiftUI menu bar content
├─ SettingsView.swift          SwiftUI Settings scene
└─ PermissionsView.swift       Onboarding for Input Monitoring
```
